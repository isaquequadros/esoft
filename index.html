O mapa da Engenharia de Software

Um guia para os estudantes que ainda não escolheram uma graduação!

Aqui estão algumas disciplinas que serão abordadas ao longo da graduação..

FUNDAMENTOS E ARQUITETURA DE COMPUTADORES


É a disciplina que estuda a estrutura interna e o funcionamento dos computadores.
Ela é essencial para o entendimento de como os computadores funcionam
e para o desenvolvimento de softwares eficientes.


ALGORITMOS E LÓGICA DE PROGRAMAÇÃO I


É uma disciplina fundamental para os cursos de Ciência da Computação e Engenharia de Software.
Ela é responsável por ensinar os conceitos básicos da lógica de programação e da construção de algoritmos.


ALGORITMOS E LÓGICA DE PROGRAMAÇÃO II


É a continuação da disciplina de Algoritmos e Lógica de Programação I.
Ela aprofunda os conceitos aprendidos na primeira disciplina
e apresenta novos tópicos como: estrutura de dados, manipulação de dados e algoritmos avançados.

ENGENHARIA DE SOFTWARE


A engenharia de software é uma área da engenharia e da computação voltada à especificação, desenvolvimento,
manutenção e criação de software, com a aplicação de tecnologias e práticas de gerência de projetos e outras disciplinas,
visando organização, produtividade e qualidade.


ESTATÍSTICA


A disciplina estatística é uma área da matemática que estuda a coleta, organização, análise e interpretação de dados. Ela é uma ferramenta essencial para a tomada de decisões em diversas áreas do conhecimento, como economia, saúde, educação, engenharia, etc.


ANÁLISE E PROJETO ORIENTADO A OBJETOS


Análise e Projeto Orientado a Objetos (APOO) é uma disciplina de Engenharia de Software que visa ensinar aos alunos como aplicar os conceitos de orientação a objetos para modelar e projetar sistemas de software. A disciplina geralmente é dividida em duas partes: análise e projeto. A análise é responsável por entender os requisitos do sistema, enquanto o projeto é responsável por definir a arquitetura e o design do sistema. Na análise, os alunos aprendem a identificar os atores e casos de uso do sistema. Os atores são os usuários do sistema, enquanto os casos de uso são as atividades que os atores realizam.

ENGENHARIA DE REQUISITOS


A disciplina de Engenharia de Requisitos é responsável por definir, documentar e gerenciar os requisitos de um sistema de software. Os requisitos são as características e funcionalidades que o sistema deve ter para atender às necessidades dos usuários e do negócio.


LÓGICA PARA COMPUTAÇÃO


A disciplina Lógica para Computação é uma disciplina introdutória à lógica matemática, com foco nas aplicações da lógica na ciência da computação. O objetivo da disciplina é fornecer aos estudantes uma compreensão dos conceitos básicos da lógica matemática, bem como suas aplicações em áreas como a teoria da computação, a inteligência artificial e a programação.


MODELAGEM DE SOFTWARE


A disciplina de Modelagem de Software é uma das principais disciplinas do curso de Engenharia de Software. Ela é responsável por ensinar aos estudantes como construir modelos que representam os requisitos, a arquitetura e o comportamento de sistemas de software. A modelagem de software é uma atividade essencial no desenvolvimento de software, pois ajuda a visualizar o sistema, a comunicar os requisitos aos stakeholders e a garantir a qualidade do sistema.

ESTRUTURA DE DADOS I


A disciplina Estrutura de Dados I é uma disciplina obrigatória dos cursos de Ciência da Computação, Engenharia de Software e áreas afins. O objetivo da disciplina é introduzir os conceitos básicos de estruturas de dados, que são mecanismos de organização e armazenamento de dados de forma eficiente.


ESTRUTURA DE DADOS II


A disciplina Estrutura de Dados II é uma disciplina obrigatória para os cursos de Ciência da Computação, Engenharia da Computação e Sistemas de Informação. Ela é uma continuação da disciplina Estrutura de Dados I, e tem como objetivo estudar as principais estruturas de dados não triviais, como árvores balanceadas, tabelas de dispersão (hashing) e representações de grafos.


PARADIGMAS DE LINGUAGEM DE PROGRAMAÇÃO


A disciplina Paradigmas de Linguagem de Programação é uma disciplina obrigatória dos cursos de graduação em Ciência da Computação e Engenharia da Software. Seu objetivo é apresentar os principais paradigmas de programação, as principais construções e características de cada um, e como eles podem ser aplicados na resolução de problemas.

PROJETO, IMPLEMENTAÇÃO E TESTE DE SOFTWARE


A disciplina PROJETO, IMPLEMENTAÇÃO E TESTE DE SOFTWARE é uma disciplina obrigatória do curso de Engenharia de Software. Ela tem como objetivo ensinar aos alunos os conceitos e técnicas fundamentais para o desenvolvimento de software de qualidade.


CÁLCULO DIFERENCIAL E INTEGRAL


A disciplina de Cálculo Diferencial e Integral é um ramo da matemática que estuda as taxas de variação de grandezas e a acumulação de quantidades. É uma disciplina fundamental para a compreensão de muitos conceitos científicos e tecnológicos, e é essencial para o estudo de áreas como Física, Química, Biologia, Engenharia e Economia.


SISTEMAS OPERACIONAIS


A disciplina de Sistemas Operacionais é uma disciplina fundamental no curso de Ciência da Computação. Ela estuda os sistemas operacionais, que são softwares que gerenciam os recursos de hardware e software de um computador. Os sistemas operacionais são responsáveis por permitir que os usuários interajam com o computador e executem programas.

PESQUISA OPERACIONAL


A Pesquisa Operacional (PO) é uma área do conhecimento que estuda, desenvolve e aplica métodos analíticos avançados para auxiliar na tomada de melhores decisões nas mais diversas áreas de atuação humana.


QUALIDADE DE SOFTWARE


A qualidade de software é uma disciplina da engenharia de software que se concentra na garantia de que o software atende às necessidades dos usuários e aos requisitos especificados. É um processo de prevenção, detecção e remoção de defeitos, bem como de melhoria da qualidade do software. A qualidade de software é importante porque pode ajudar a garantir que o software seja confiável, eficiente, seguro e fácil de usar. Isso pode levar a uma maior satisfação do usuário, a um aumento da produtividade e a uma redução dos custos de desenvolvimento e manutenção de software.


GERENCIAMENTO DE SOFTWARE


A disciplina de Gerência de Software é uma disciplina que ensina os fundamentos de gerenciamento de projetos de software. Ela é essencial para qualquer profissional que queira trabalhar na área de desenvolvimento de software, pois fornece as ferramentas e técnicas necessárias para gerenciar projetos de software de forma eficaz. A disciplina de Gerência de Software geralmente é ministrada em cursos de graduação e pós-graduação em Ciência da Computação, Engenharia da Computação e Sistemas de Informação. Ela também é oferecida em cursos de extensão e MBAs.

BANCO DE DADOS I


A disciplina Banco de Dados I é uma disciplina introdutória aos conceitos básicos de bancos de dados. Ela é geralmente oferecida no primeiro ano de cursos de Ciência da Computação, Engenharia da Computação e Engenharia de Software.


BANCO DE DADOS II


Banco de Dados II é uma disciplina que aprofunda os conhecimentos em modelagem, projeto e implementação de sistemas de banco de dados. Durante as aulas, os alunos aprendem sobre técnicas avançadas de modelagem de dados, normalização, otimização de consultas, transações e controle de concorrência.


TÓPICOS EM COMPUTAÇÃO I

tem como objetivo aprofundar os conhecimentos dos alunos em temas específicos da computação. O conteúdo da disciplina varia de acordo com a instituição de ensino, mas geralmente inclui tópicos como: Inteligência artificial Aprendizado de máquina Visão computacional Redes neurais Segurança da informação Big data Computação em nuvem Computação gráfica A disciplina é ministrada em formato de aulas teóricas e práticas. As aulas teóricas abordam os conceitos e fundamentos teóricos dos tópicos estudados. As aulas práticas permitem que os alunos apliquem os conhecimentos adquiridos em situações reais.

PROGRAMAÇÃO DE SISTEMAS I


A disciplina de Programação de Sistemas I é uma disciplina introdutória à programação de computadores. Ela apresenta os conceitos básicos da programação, como variáveis, tipos de dados, operadores, controle de fluxo, funções e estruturas de dados. Também aborda os conceitos de orientação a objetos, que é um paradigma de programação que permite a criação de programas mais complexos e reutilizáveis.


PROGRAMAÇÃO DE SISTEMAS II


A disciplina Programação de Sistemas II é uma continuação da disciplina Programação de Sistemas I. Ela visa aprofundar os conhecimentos dos alunos em programação de computadores, focando em tópicos como: Estruturas de dados avançadas Algoritmos avançados Sistemas operacionais Redes de computadores Programação paralela e distribuída


DESIGN E INTERAÇÃO


A disciplina de Design e Interação é uma área do design que foca na criação de interfaces e experiências de usuário. O objetivo do design de interação é criar produtos e serviços que sejam fáceis de usar, agradáveis e que atendam às necessidades dos usuários. A disciplina abrange uma ampla gama de tópicos, incluindo: Usabilidade: Como garantir que os produtos e serviços sejam fáceis de usar e que os usuários possam realizar suas tarefas com eficiência. Experiência do usuário: Como criar produtos e serviços que sejam agradáveis de usar e que criem uma conexão emocional com os usuários. Design centrado no usuário: Como colocar o usuário no centro do processo de design e entender suas necessidades e expectativas.

TÓPICOS ESPECIAIS EM ENGENHARIA DE SOFTWARE I


Tópicos Especiais em Engenharia de Software I é uma disciplina que aborda temas específicos e atuais na área de Engenharia de Software. A disciplina é oferecida eventualmente com o objetivo de suprir necessidades de formação específicas e indispensáveis dos alunos e tratar de temas emergentes e/ou inovadores na área de Engenharia de Software.


TRABALHO DE CONCLUSÃO DE CURSO I

O Trabalho de Conclusão de Curso I (TCC I) é uma disciplina obrigatória para alunos do último ano de graduação em Engenharia de Software.


TÓPICOS EM COMPUTAÇÃO II

A disciplina Tópicos em Computação II é uma disciplina optativa oferecida para alunos de graduação em Ciência da Computação e Engenharia de Software. O objetivo da disciplina é proporcionar aos alunos uma visão aprofundada de tópicos específicos da área de computação, que não são abordados em outras disciplinas do currículo básico.

FUNDAMENTOS DE REDES DE COMPUTADORES


Fundamentos de Redes de Computadores é uma disciplina introdutória ao estudo de redes de computadores. Ela é oferecida em cursos de graduação e pós-graduação em Ciência da Computação, Engenharia de Software e outras áreas relacionadas à tecnologia da informação. O objetivo da disciplina é fornecer aos alunos uma compreensão básica dos conceitos e tecnologias envolvidos em redes de computadores. Os tópicos abordados incluem: Princípios básicos de redes de computadores Arquitetura de redes de computadores Tipos de cabeamentos de redes Principais protocolos de rede e suas funções Principais dispositivos de rede e suas funcionalidades


GESTÃO DE PROJETOS


A disciplina de Gestão de Projetos é uma área do conhecimento que se concentra no planejamento, execução, monitoramento e controle de projetos. Um projeto é um esforço temporário empreendido para criar um produto, serviço ou resultado único. A gestão de projetos é importante porque ajuda a garantir que os projetos sejam bem-sucedidos. Isso significa que os projetos são concluídos dentro do prazo, do orçamento e de acordo com os requisitos do cliente.


SEGURANÇA E AUDITORIA DE SISTEMAS


A disciplina Segurança e Auditoria de Sistemas é uma disciplina obrigatória dos cursos de graduação em Sistemas de Informação e Engenharia da Computação. O objetivo da disciplina é capacitar os alunos para compreender os conceitos e as técnicas de segurança da informação e auditoria de sistemas, bem como para aplicar estes conhecimentos para o desenvolvimento e a avaliação de sistemas seguros. A disciplina é dividida em duas partes: Segurança da Informação e Auditoria de Sistemas. A parte de Segurança da Informação aborda os conceitos básicos de segurança da informação, como ameaças, vulnerabilidades, controles de segurança e políticas de segurança. A parte de Auditoria de Sistemas aborda os conceitos básicos de auditoria de sistemas, como o processo de auditoria, os tipos de auditoria e as ferramentas de auditoria.


TÓPICOS ESPECIAIS EM ENGENHARIA DE SOFTWARE II


A disciplina Tópicos Especiais em Engenharia de Software II Ela tem como objetivo aprofundar o conhecimento dos alunos em áreas específicas da engenharia de software. A disciplina é ministrada por um professor convidado, que é um especialista na área escolhida. Os tópicos abordados na disciplina variam de acordo com o professor convidado, mas geralmente incluem temas como: Teste de software Qualidade de software Segurança de software Engenharia de requisitos Engenharia de arquitetura de software Engenharia de testes de software


TRABALHO DE CONCLUSÃO DE CURSO IITRABALHO DE CONCLUSÃO DE CURSO II


A disciplina Trabalho de Conclusão de Curso II (TCC II) O objetivo da disciplina é orientar os alunos na elaboração do Trabalho de Conclusão de Curso (TCC), que é um trabalho de pesquisa científica que deve ser apresentado como requisito para a obtenção do diploma de bacharel em Engenharia de Software. A disciplina é dividida em duas etapas: Etapa I: Apresentação e aprovação do Projeto de Pesquisa. Etapa II: Elaboração e apresentação do TCC.
